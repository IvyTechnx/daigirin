## コミット

### 基本

```
コミットして。
```

Claude Codeが差分を見て、適切なコミットメッセージを自動で書いてくれる。

### メッセージ指定

```
「ヘッダーにダークモード切替を追加」でコミットして。
```

### 変更を分けてコミット

```
今の変更をファイルごとに分けてコミットして。
UI系とロジック系で分けて。
```

## ブランチ

### 新しいブランチで作業開始

```
feature/dark-mode ブランチを作って切り替えて。
```

### ブランチ整理

```
マージ済みのブランチを一覧して。
削除していいか確認して。
```

## PR（Pull Request）

### PR作成

```
今のブランチでPR作って。
変更内容を自動でサマリーして。
```

### 丁寧なPR

```
PRを作って。
- タイトルは短く
- 概要に変更点を箇条書き
- テスト計画も書いて
```

### PR作成からpushまで一気に

```
今の変更をコミットして、リモートにpushして、PR作って。
一気にやって。
```

## レビュー

### 自分のコードのレビュー

```
今の差分をレビューして。
バグ、パフォーマンス、セキュリティの観点で。
```

### 他人のPRレビュー

```
このPRをレビューして。
https://github.com/xxxx/xxxx/pull/123
気になる点をコメントして。
```

（GitHub MCPサーバーが必要）

## コンフリクト解消

```
コンフリクトが起きた。直して。
mainブランチの変更を優先で。
```

## やらかしたとき

### 直前のコミットを取り消したい

```
さっきのコミット取り消して。変更はステージに残して。
```

### 特定のファイルだけ戻す

```
src/app/page.tsx を前のコミットの状態に戻して。
```

### 間違ってmainに直接コミットした

```
さっきのコミットを新しいブランチに移して、
mainは元に戻して。
```
