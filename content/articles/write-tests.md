## 基本

```
このプロジェクトにテストを追加して。
主要な関数のユニットテストを書いて。
```

Claude Codeがテストフレームワーク(vitest/jest)の導入からテストコード作成まで全部やってくれる。

## 特定のファイルのテスト

```
src/lib/utils.ts のテストを書いて。
全関数カバーして、エッジケースも含めて。
```

## 既存テストが落ちてる

```
テスト実行して。落ちてるやつがあったら直して。
テストコードが間違ってるのか、実装が間違ってるのか判断して。
```

## 最強プロンプト: 全部通るまでループ

```
npm test を実行して。
失敗したテストがあれば原因を調べて修正して。
全テストがパスするまで繰り返して。
```

Claude Codeが自分でテスト→失敗確認→修正→再テストのサイクルを回してくれる。

## テストの種類別

### APIルートのテスト

```
src/app/api/ 以下のAPIルートのテストを書いて。
成功ケースとエラーケース（400, 404, 500）を含めて。
```

### コンポーネントのテスト

```
src/components/article-card.tsx のテストを書いて。
レンダリングされること、propsが反映されること、
リンク先が正しいことを確認して。
```

### E2Eテスト

```
Playwrightでトップページの E2Eテストを書いて。
- ページが表示される
- ナビゲーションが機能する
- 記事をクリックして詳細ページに遷移する
```

## テストを書かずに確認させる

### 手動確認プロンプト

```
この変更で壊れてるところないか確認して。
ビルドが通ることと、主要な画面が表示されることを確認して。
```

### 型チェック

```
tsc --noEmit で型エラーがないか確認して。
あれば全部直して。
```

### lint

```
lint通して。警告も含めて全部直して。
```

## CI用の設定

```
GitHub Actionsでプッシュ時にテストが走るようにして。
.github/workflows/test.yml を作って。
```
